# راهنمای کامل راه‌اندازی (گیت‌هاب + Railway)

این پروژه از **دو سرویس** تشکیل شده که هرکدوم جدا روی Railway بالا میان:

1. **ربات تلگرام** (`bot.py`) — همیشه روشنه، با polling به تلگرام وصله، پیام‌ها رو جواب می‌ده.
2. **سرورِ بازیِ حکم** (`hokm_server/`) — یه وب‌سرویسِ کوچیک (FastAPI) که صفحه‌ی وبِ بازیِ
   حکمِ چهارنفره رو سرو می‌کنه. ربات با دستورِ `/hokm` یه لینک به این سرور می‌فرسته که با
   دکمه‌ی WebApp داخلِ خودِ تلگرام باز می‌شه.

اگه بازیِ حکم رو نمی‌خوای، فقط سرویسِ ۱ (ربات) کافیه.

---

## قدم ۱: ساختنِ ربات در تلگرام و گرفتنِ TOKEN

1. توی تلگرام برو سراغِ [@BotFather](https://t.me/BotFather).
2. بفرست `/newbot` و اسم و یوزرنیم دلخواه رو بده.
3. یه توکن مثلِ `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11` بهت می‌ده. این می‌شه مقدارِ
   متغیرِ **`TOKEN`**.
4. اگه می‌خوای ربات توی گروه‌ها ادمین‌کارها (بن/میوت/پاک‌سازی) رو انجام بده، حتماً خودِ
   ربات رو توی گروه **ادمین** کن.
5. برای این‌که ری‌اکشن و حذفِ پیام و... کار کنه، از `/mybots` → انتخابِ ربات → `Bot Settings`
   → `Group Privacy` رو **Disable** کن (تا ربات بتونه همه‌ی پیام‌های گروه رو ببینه، نه فقط
   دستورها).

## قدم ۲: گرفتنِ کلیدهای هوش مصنوعی (API Key)

هرچی بیشتر از این‌ها رو ست کنی، ربات کمتر با «سقفِ رایگان تموم شد» مواجه می‌شه، چون
سهمیه‌ی همه با هم جمع می‌شه. سه‌تای اول اجباری‌ان، بقیه اختیاری‌ان.

| متغیر | سرویس | لینکِ گرفتنِ کلید | اجباری؟ |
|---|---|---|---|
| `GEMINI_KEY` | Google Gemini | https://aistudio.google.com/apikey | ✅ بله |
| `GROQ_KEY` | Groq Cloud | https://console.groq.com/keys | ✅ بله |
| `CEREBRAS_KEY` | Cerebras Cloud | https://cloud.cerebras.ai/ (بخشِ API Keys) | ✅ بله |
| `OPENROUTER_KEY` | OpenRouter | https://openrouter.ai/keys | اختیاری |
| `MISTRAL_KEY` | Mistral AI | https://console.mistral.ai/api-keys | اختیاری |
| `SAMBANOVA_KEY` | SambaNova Cloud | https://cloud.sambanova.ai/apis | اختیاری (جدید) |
| `TOGETHER_KEY` | Together AI | https://api.together.ai/settings/api-keys | اختیاری (جدید) |

نکته: همه‌ی این سرویس‌ها یه سطحِ رایگان یا اعتبارِ اولیه‌ی رایگان دارن؛ برای ثبت‌نام فقط یه
ایمیل لازمه. هیچ‌کدوم نیازی به کارتِ بانکی/اعتباریِ بین‌المللی ندارن (به‌جز احتمالاً
تاییدِ شماره‌موبایل در بعضی‌هاشون).

## قدم ۳: (اختیاری) پکِ استیکر برای وقتی هوش مصنوعی جواب نداد

1. توی تلگرام برو سراغِ [@Stickers](https://t.me/Stickers) (باتِ رسمیِ تلگرام برای ساختِ پک).
2. با `/newpack` یه پکِ استیکرِ جدید بساز (چندتا استیکرِ بامزه/دلقکی بهش اضافه کن).
3. آخرِ کار یه لینک بهت می‌ده مثلِ `t.me/addstickers/my_clown_pack_by_yourbot` — همون قسمتِ
   آخر (`my_clown_pack_by_yourbot`) رو به‌عنوانِ متغیرِ **`STICKER_PACK_NAME`** ست کن.
4. اگه این رو ست نکنی، ربات فقط یه پیامِ متنی می‌فرسته (بدونِ استیکر) وقتی هیچ سرویسی جواب نده.

## قدم ۴: بالا بردنِ کد روی گیت‌هاب

```bash
cd project
git init
git add .
git commit -m "اولین نسخه‌ی ربات + بازیِ حکم"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO_NAME.git
git push -u origin main
```

(اگه گیت نصب نیست یا نمی‌خوای از خط فرمان استفاده کنی، می‌تونی از داخلِ سایتِ
github.com یه ریپوی خالی بسازی و فایل‌ها رو مستقیم آپلود کنی — دکمه‌ی
"Add file → Upload files".)

## قدم ۵: دیپلویِ سرویسِ ربات روی Railway

1. برو roی [railway.app](https://railway.app) و با گیت‌هاب لاگین کن.
2. `New Project` → `Deploy from GitHub repo` → ریپوی همین پروژه رو انتخاب کن.
3. چون فایلِ `Procfile` توی ریشه‌ی پروژه‌ست (`worker: python bot.py`)، Railway خودش
   می‌فهمه این یه worker (بدونِ پورت/URL عمومی) هست.
4. برو تبِ **Variables** همین سرویس و همه‌ی متغیرهای بالا رو اضافه کن:
   `TOKEN`, `GEMINI_KEY`, `GROQ_KEY`, `CEREBRAS_KEY`, و هرکدوم از اختیاری‌ها که داری.
5. اگه می‌خوای دیتاها (اخطارها، لقب‌ها، یادگیری‌ها و...) بعدِ ری‌استارت از بین نره:
   از داخلِ همون پروژه‌ی Railway، `+ New` → `Database` → `Add PostgreSQL` رو بزن.
   Railway خودش متغیرِ `DATABASE_URL` رو می‌سازه و به سرویسِ ربات وصل می‌کنه (نیازی به
   کارِ دستی نیست، فقط مطمئن شو که هر دو تو یه پروژه‌ن).
6. `Deploy` رو بزن. لاگ‌ها رو چک کن؛ اگه خط "✅ ربات با موفقیت روشن شد" رو دیدی، تموم شد.

## قدم ۶: دیپلویِ سرویسِ بازیِ حکم (اختیاری، فقط اگه بازیِ حکم رو می‌خوای)

1. توی همون پروژه‌ی Railway: `+ New` → `GitHub Repo` → دوباره همون ریپو رو انتخاب کن
   (Railway اجازه می‌ده یه ریپو رو چندبار به‌عنوانِ چندتا سرویسِ جدا اضافه کنی).
2. روی این سرویسِ جدید برو `Settings` → `Root Directory` رو بذار: `hokm_server`
   (چون کدِ این بخش توی پوشه‌ی `hokm_server/` هست، نه ریشه‌ی ریپو).
3. چون `hokm_server/Procfile` می‌گه `web: uvicorn app:app ...`، Railway این یکی رو
   به‌عنوانِ **وب‌سرویس** می‌شناسه و یه دامنه‌ی عمومی (`https://xxxx.up.railway.app`) بهش
   می‌ده. اگه خودش ندادش، برو `Settings` → `Networking` → `Generate Domain`.
4. اون دامنه (مثلاً `https://hokm-production-ab12.up.railway.app`) رو کپی کن.
5. برگرد به سرویسِ **ربات** (سرویسِ اول) → `Variables` → یه متغیرِ جدید اضافه کن:
   `HOKM_WEBAPP_URL` = همون دامنه (بدونِ اسلشِ آخر).
6. سرویسِ ربات رو دوباره Deploy کن (یا صبر کن خودش رفرش کنه) تا متغیرِ جدید اعمال بشه.

بعدِ این مرحله، دستورِ `/hokm` توی تلگرام یه دکمه می‌سازه که با یه تپ، صفحه‌ی بازیِ حکم
رو *داخلِ خودِ تلگرام* (به‌شکلِ WebApp) باز می‌کنه. نیازی به تاییدِ دامنه توسطِ BotFather
نیست، چون این از نوعِ دکمه‌ی inline با `web_app` هست (نه Menu Button سراسری).

> نکته: چون هر بار `/hokm` یه کدِ اتاقِ رندوم می‌سازه، ۴ نفر باید روی **همون** دکمه‌ای که
> ربات فرستاده تپ کنن تا سرِ یه میز بشینن؛ اگه یکی جدا `/hokm` بزنه، میزِ جدیدی می‌سازه.

## قدم ۷: تنظیماتِ اختیاریِ دیگه

| متغیر | توضیح |
|---|---|
| `BOT_CALL_NAME` | یه اسمِ اختصاصی برای ربات (مثلاً `کینگر`) تا وقتی کاربرها این اسم رو صدا بزنن هم بفهمه. |
| `WAITING_STICKER_ID` | فایل‌آی‌دیِ یه استیکر که به‌جایِ پیامِ «صبر کن...» نشون داده بشه. |
| `WAITING_GIF_URL` | لینکِ یه گیف به‌جایِ همون پیامِ انتظار. |
| `DATABASE_URL` | خودکار توسطِ افزونه‌ی Postgres روی Railway ست می‌شه؛ دستی لازم نیست. |

## سوالِ متداول

**«چرا کریپتو/قیمتِ دلار حذف شد؟»** — این دو ویژگی از اول توی این نسخه از کد نبودن؛ اگه
جایی نسخه‌ی قدیمی‌تری داشتی که این‌ها رو داشت، همینجا حذف موندن، هیچ چیزِ اضافه‌ای برای
پاک‌کردن نمونده.

**«چطور یه سرویسِ جدیدِ هوش مصنوعی دیگه اضافه کنم؟»** — اگه سرویس API سازگار با OpenAI
داره (اکثرِ سرویس‌های جدید همین‌طورن)، توی `bot.py` دنبالِ الگوی `sambanova_client`/
`together_client` بگرد و دقیقاً همون الگو رو با کلید و base_url جدید کپی کن، بعد اسمِ
تابعش رو به `TEXT_PROVIDERS` (و در صورتِ پشتیبانیِ عکس، به `IMAGE_PROVIDERS`) اضافه کن.
